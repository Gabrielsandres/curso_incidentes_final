  add column if not exists video_external_id  text        null,
  add column if not exists workload_minutes   integer     null

  where deleted_at is not null;

Section 5 — pending_enrollments table (D-10 Option A):
```sql
create table if not exists public.pending_enrollments (
  id          uuid        primary key default gen_random_uuid(),
  email       text        not null,
  course_id   uuid        not null references public.courses(id) on delete cascade,
  invited_by  uuid        null references public.profiles(id) on delete set null,
  expires_at  timestamptz null,
  created_at  timestamptz not null default now(),
  constraint pending_enrollments_email_check
    check (email ~* '^[^@\s]+@[^@\s]+\.[^@\s]+$')
);

create index if not exists idx_pending_enrollments_email
  on public.pending_enrollments (lower(email));

create index if not exists idx_pending_enrollments_course_id
  on public.pending_enrollments (course_id);

alter table public.pending_enrollments enable row level security;
```
Add RLS policies from RESEARCH.md Q4: "Admins manage pending enrollments" (authenticated, profile.role = 'admin', USING + WITH CHECK), and "Service role manages pending enrollments" (service_role, using true, with check true). Use drop policy if exists before each create policy (idempotent pattern).

Section 6 — courses RLS for students (CAT-05, CAT-07, T4):
```sql
drop policy if exists "Authenticated users read published courses" on public.courses;
create policy "Authenticated users read published courses"
  on public.courses
  for select to authenticated
  using (
    (published_at is not null and archived_at is null)
    or exists (
      select 1 from public.profiles p
      where p.id = auth.uid() and p.role = 'admin'
    )
  );
```

|-----------|----------|-----------|-------------|-----------------|
| T-02-T3 | Information Disclosure | lessons.deleted_at (query-level filter) | mitigate | Soft-deleted lessons filtered via `.is("deleted_at", null)` in student-facing queries (getCourseWithContent); admin queries omit filter intentionally. Tests in plan 02-03 verify filter presence. |
| T-02-T4 | Information Disclosure | courses.archived_at (RLS policy) | mitigate | Migration adds RLS policy "Authenticated users read published courses" filtering `archived_at IS NULL AND published_at IS NOT NULL` for non-admins. Admin bypasses via OR clause in policy. |
