### 1. LessonPlayer — Provider Abstraction + Watermark Overlay

**Location:** `src/components/course/lesson-player.tsx`

**Props change:** Replace `lesson: LessonWithMaterials` + `initialIsCompleted` with:
```ts
type LessonPlayerProps = {
  embedUrl: string;             // signed URL resolved server-side by RSC
  provider: "youtube" | "bunny";
  watermarkText: string | null; // null = no watermark (YouTube dev mode)
  lessonId: string;
  lessonTitle: string;
  lessonDescription: string | null;
  initialIsCompleted: boolean;
};
```
The RSC (`page.tsx`) calls `VideoProvider.getPlayableSource()` and passes results as props. LessonPlayer becomes a pure presentational + event-wiring component.

**Video container (unchanged shell):**
```
class: aspect-video overflow-hidden rounded-xl border border-slate-200 bg-black
```
Position: `relative` — required to contain the absolute watermark overlay.
```
class: aspect-video overflow-hidden rounded-xl border border-slate-200 bg-black relative
```

**Iframe (Bunny and YouTube fallback):**
```tsx
<iframe
  title={lessonTitle}
  src={embedUrl}
  className="h-full w-full"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
  allowFullScreen
  loading="lazy"
/>
```
Both providers use a simple `<iframe>` — YouTube IFrame API JavaScript is removed. Auto-completion for Bunny is handled via `window.postMessage`; YouTube auto-completion is handled by adding `&enablejsapi=1` to the embed URL and listening for `postMessage` events.

**Watermark overlay (Bunny prod only):**

Renders when `watermarkText !== null`. Absolutely positioned over the video container.

```tsx
{watermarkText !== null ? (
  <WatermarkOverlay text={watermarkText} />
) : null}
```

`WatermarkOverlay` is a small inline component (co-located in `lesson-player.tsx`, not a separate file):
```tsx
function WatermarkOverlay({ text }: { text: string }) {
  const [corner, setCorner] = useState<0 | 1 | 2 | 3>(0);

  useEffect(() => {
    const interval = setInterval(() => {
      setCorner((c) => ((c + 1) % 4) as 0 | 1 | 2 | 3;
    }, 30000);
    return () => clearInterval(interval);
  }, []);

  const positionClass = [
    "top-3 left-3",
    "top-3 right-3",
    "bottom-12 right-3",   // bottom-12 to avoid player controls overlap
    "bottom-12 left-3",
  ][corner];

  return (
    <div
      aria-hidden="true"
      className={`pointer-events-none absolute ${positionClass} select-none text-sm font-medium text-white transition-opacity duration-500`}
      style={{ opacity: 0.12 }}
    >
      {text}
    </div>
  );
}
```

Constraints:
- `pointer-events: none` — CONTEXT.md D-08, AP-01
- `aria-hidden="true"` — decorative overlay; should not be read by screen readers
- `select-none` — prevents text selection (reduces trivial copy attacks)
- `opacity: 0.12` — mid-point of CONTEXT.md D-07 range (10–15%); expressed as inline style not Tailwind opacity (avoids PurgeCSS stripping an unusual fraction)
- `transition-opacity duration-500` — smooth fade when position changes (500ms)
- Corner positions cycle: top-left → top-right → bottom-right → bottom-left (CONTEXT.md D-05)
- `bottom-12` on bottom positions — leaves space above Bunny player controls (approx 48px footer)

**Video unavailable state (unchanged pattern):**
```
class: rounded-xl border border-dashed border-red-200 bg-red-50 p-4 text-sm text-red-700
copy: "Não foi possível carregar o vídeo desta aula. Verifique se o ID de vídeo salvo é válido."
```
Trigger: when `embedUrl` is empty/null (should not happen in normal flow — RSC guards this).

**Bunny auto-completion (postMessage):**
`useEffect` in LessonPlayer listens for `window.postMessage` events filtered by `event.data?.event === "ended"` from the Bunny embed origin (`iframe.mediadelivery.net`). Calls `markLessonAsCompleted("video-end")` on match. Manual button remains always visible (CONTEXT.md D-13).

**YouTube auto-completion fallback:**
Append `&enablejsapi=1` to the YouTube embed URL. Listen for `postMessage` where `event.data` is a JSON string with `event: "infoDelivery"` and `info.playerState === 0` (ENDED). No external YouTube IFrame API script loaded — postMessage approach is simpler and avoids the global `window.YT` dependency.

Note: The YouTube IFrame API global (`window.YT`) and its loader (`loadYouTubeIframeApi`) are removed from the file in this refactor.

Every interactive element must cover all states. No new state colors introduced.

| Element | Default | Hover | Disabled | Focus |
|---------|---------|-------|----------|-------|
| "Salvar aula" button (sky-600) | bg-sky-600 white text | bg-sky-700 | opacity-70 cursor-not-allowed | focus-visible:outline-sky-500 |
| Provider `<select>` | border-slate-300 bg-white | — | — | focus:border-sky-500 focus:ring-2 focus:ring-sky-100 |

| Watermark overlay | opacity: 0.12, pointer-events:none | n/a (pointer-events:none) | n/a | aria-hidden (not focusable) |
| Video iframe | Renders embedded content | — | — | Standard browser iframe focus |
| Video unavailable error block | Static — no interaction | — | — | — |

**Watermark position transition:** When the 30-second interval fires and `corner` state changes, the overlay transitions with `transition-opacity duration-500` (500ms CSS transition). The text fades out then in using React's re-render with the transition applied to opacity. Implementation: toggle opacity from 0.12 to 0 on interval start, update corner, then fade back to 0.12 after a 100ms delay.

Simpler alternative (acceptable): no opacity transition — just move positions every 30s without animation. This is Claude's discretion per CONTEXT.md. The transition adds polish without complexity.

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Watermark not read by screen readers | `aria-hidden="true"` on watermark overlay `<div>` |
| Watermark not keyboard-focusable | `pointer-events: none` + no `tabIndex`; overlay is excluded from tab order |
| Watermark not selectable | `select-none` Tailwind class (`user-select: none`) |
| Video iframe — accessible title | `title={lessonTitle}` on `<iframe>` — existing pattern, preserved |
| Completion button states announced | `disabled` attribute changes button label text; screen readers read updated label |
| Provider select — label association | `<label>` wrapping the `<select>` — standard pattern used throughout admin forms |
| Video ID input — label association | `<label>` wrapping the `<input>` — same pattern |
| Field validation errors | `<p className="text-xs text-red-600">` positioned immediately after the field; associated via DOM order |
| No `aria-live` needed for watermark rotation | Decorative element; position change should not be announced |
