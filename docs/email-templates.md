# Supabase Auth Email Templates — Source of Truth

> **CRITICAL:** This doc is the canonical source of truth for the pt-BR email
> templates configured in **Supabase Dashboard → Authentication → Email Templates**.
> Supabase has no API to push templates programmatically (verified via Pitfall 2 in
> Phase 5 RESEARCH.md — as of 2026 the templates live only in panel state). Whenever
> the templates in this file change, the operator MUST manually re-paste into the
> Supabase Dashboard for the change to take effect.
>
> Drift detection is **manual** — see "Verification" at the bottom.

---

## Setup steps (Invite User template)

1. Sign in to the Supabase Dashboard for the **MDHE** project.
2. Navigate to **Authentication → Email Templates**.
3. Open the **Invite User** template.
4. Replace the **Subject** field with the value in §Subject below.
5. Switch to the HTML editor and replace the body with the value in §HTML Body below.
6. Click **Save**.
7. Send a smoke-test invite (see §Verification) and confirm the email contains pt-BR copy with the institution name interpolated.

**Notes on the template:**

- `{{ .Data.institution_name }}` resolves from the `data: { institution_name }` field passed to `auth.admin.inviteUserByEmail` by the `Criar-usuario` Edge Function (D-09 + D-11). Phase 5's institutional invite path always populates this metadata.
- `{{ if .Data.institution_name }} ... {{ else }} ... {{ end }}` is Go-template syntax (Supabase Auth uses Go's `html/template` engine). The empty-string branch handles legacy B2C invites from `/admin/usuarios` that do not pass `institution_name` (preserves backward compatibility per D-10 fallback requirement).
- Supabase Auth's Go template engine HTML-escapes `{{ .Data.* }}` interpolations by default — institution names containing `<` or `>` are rendered safely (verified default behavior; Threat Model T-05-04-04).
- The button link `{{ .ConfirmationURL }}` is generated by Supabase Auth and includes the secure invite token. Do NOT replace it.

## Verification

After updating the template in the Supabase Dashboard, verify:

1. **Subject inspection** — open the saved Invite User template; confirm the Subject field exactly matches §Subject above (no extra spaces, correct quotes).
2. **Body inspection** — confirm the HTML body matches §HTML Body, especially the `{{ if .Data.institution_name }} ... {{ else }} ... {{ end }}` block (Supabase panel sometimes auto-strips whitespace; the Go syntax must be intact).
3. **Smoke test (institutional path)** — from `/admin/instituicoes/[slug]`, invite a personal test email with an institution selected; confirm the received email:
   - Subject contains the institution name (e.g., "Bem-vindo(a) à plataforma MDHE — convite de Colégio Marista")
   - Body contains "como aluno(a) da Colégio Marista"
   - "Aceitar convite e criar minha senha" button is present and links to a Supabase confirmation URL
4. **Smoke test (B2C fallback path)** — from `/admin/usuarios` (legacy non-institutional invite), invite a personal test email; confirm:
   - Subject still resolves (no `{{ .Data.institution_name }}` literal in the rendered subject — it should be EMPTY or render as a no-name fallback subject in the panel — verify behavior; if subject literal is empty string, document as known limitation in §Notes and add a fallback subject for B2C below)
   - Body contains the `{{ else }}` branch text: "Você foi convidado(a) pela MDHE Consultoria para acessar a plataforma de cursos." (no institution name)
5. **Drift detection (manual, on every release):** before each production deploy, compare the panel template against this file's content. If they differ, re-paste from this file (single source of truth).

## Known limitations

- **Subject does not support `{{ if }}` Go-template fallback** in Supabase Auth (subjects are evaluated as a single template string). When `institution_name` is absent (legacy B2C invite), the subject resolves to "Bem-vindo(a) à plataforma MDHE — convite de " (trailing space). For v1 this is acceptable since `/admin/usuarios` is the only path that triggers the no-institution case and it's an internal admin tool. **If polish is needed before user-facing release**, the operator can craft a separate "Magic Link" template for B2C invites — captured as deferred idea.
- **Template drift** is detected manually — see Pitfall 2 in `.planning/phases/05-b2b-institution-manager/05-RESEARCH.md`.
